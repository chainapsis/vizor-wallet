# Vizor Wallet v0.0.30

This release adds token holder voting to Vizor, plus broader secret passphrase
import support, swap address editing improvements, text field message
improvements, and sync performance updates.

## What's New

- Added a Vote section for token holder voting, including voting rounds, vote
  details, voting power checks, answer review, submission progress, receipts,
  and results.
- Added Keystone support for voting with QR-based signing and multi-bundle
  signing progress.
- Added voting config settings so advanced users can inspect, copy, save, and
  switch voting config sources.

## Improvements

- Secret passphrase imports now support valid 12, 15, 18, 21, and 24-word
  phrases.
- Voting shows eligibility status based on the voting round snapshot block,
  including clear guidance when an account does not have enough eligible
  shielded funds.
- Voting can wait for wallet sync through the voting round snapshot block and
  continue once the wallet is ready.
- Voting rounds and results can show ZIP metadata and forum discussion links
  when provided.
- Swap refund and recipient address editing now supports saved labels and
  avatars, EVM contact reuse across EVM chains, and clearer duplicate-address
  feedback.
- Long text field status messages now show the full message in a tooltip when
  truncated.
- Reduced unnecessary wallet repair checks during sync.

## Notes

- Voting eligibility is based on the voting round snapshot block and requires at
  least 5 eligible shielded notes totaling 0.125 ZEC.
- If Vizor is generating voting proofs or waiting for sync during submission,
  keep the app open until submission completes.
