9. Background Sync

Background sync는 iOS와 Android 모두 지원한다. iOS는 BGContinuedProcessingTask + C FFI로 Rust sync를 직접 실행하고, Android는 flutter_foreground_task를 통해 notification만 관리하며 sync 자체는 동일한 FRB Stream으로 실행된다.

9.1 iOS BGContinuedProcessingTask (iOS 26+)

핵심 원리: Dart 코드는 iOS background에서 실행할 수 없다. BGContinuedProcessingTask의 핸들러는 Swift에서 실행되므로, Rust를 C FFI로 직접 호출해야 한다.

Mode 전환 메커니즘:

Rust에 DESIRED_SYNC_MODE AtomicU8이 존재한다 (0=none, 1=foreground, 2=background). 이 값으로 foreground/background sync 간 전환을 제어한다.

  • Foreground → Background: Dart가 setSyncMode(2) 호출 → Rust fg sync가 다음 batch에서 mode mismatch 감지 후 종료 → Swift BGTask handler가 while(mode!=2 || is_running) sleep(2s)로 대기 → 조건 충족 시 C FFI zcash_run_full_sync() 호출로 bg sync 시작
  • Background → Foreground: Dart가 setSyncMode(1) 호출 → Rust bg sync가 다음 batch에서 mode mismatch 감지 후 종료 → Dart가 while(isSyncRunning()) await 200ms로 대기 → fg sync 시작
  • Expiration: expirationHandler가 mode=0 + cancel 설정 → resubmit 하지 않음 → 앱이 foreground로 돌아오면 AppLifecycleListener.onResume에서 mode==0 감지 → foreground sync 재시작
  • BGTask resubmit: sync가 종료되었는데 mode가 여전히 2이면 BGTask를 다시 제출하여 background sync 계속
Dart: "Sync on Background" 버튼 탭 → setSyncMode(2) [FRB] → Rust fg sync: mode mismatch → 종료 → MethodChannel → Swift: BGContinuedProcessingTaskRequest 제출 → OS가 핸들러 실행 (background queue) → while(mode!=2 || is_running) sleep(2s) [대기] → zcash_run_full_sync() [C FFI, blocking] → tokio runtime 생성 → run_sync_inner() [gRPC + scan + enhancement] → C callback → task.progress 업데이트 (NSProgress, thread-safe) → C callback → SyncProgressStreamHandler → EventChannel → Dart (앱이 foreground면) → sync 종료, mode==2이면 BGTask resubmit → task.setTaskCompleted(success:)

9.2 Android Foreground Service

Android에서는 flutter_foreground_task를 사용하여 notification만 관리한다. Sync 자체는 foreground와 동일한 Dart FRB Stream으로 실행되며, mode 전환 메커니즘은 없다.

  • enableBackgroundSync(): foreground service notification을 추가한다. 사용자에게 sync가 진행 중임을 알리고 시스템이 앱을 kill하지 않도록 한다.
  • disableBackgroundSync(): notification을 제거한다. Sync 자체는 중단되지 않는다.
