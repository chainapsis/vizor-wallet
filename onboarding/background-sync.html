9. Ironwood 백그라운드 작업

일반 지갑 sync는 Dart → FRB → Rust foreground 경로 하나만 사용한다. 예전에 존재하던 iOS 일반 background sync와 Android notification delegate는 제거되었다. iOS native background task는 Ironwood migration 준비와 예약 전송에만 사용한다.

9.1 Denomination preparation (iOS 26+)

BackgroundMigrationPreparationManager는 com.keplr.vizor.ironwood-preparation BGContinuedProcessingTask를 관리한다. Swift는 Dart isolate 없이도 동작해야 하므로 rust/src/ffi.rs의 migration 전용 C FFI를 호출한다.

BGContinuedProcessingTask → migration preparation operation begin → foreground sync가 SYNC_RUNNING을 놓을 때까지 대기 → preparation-only sync (gRPC + scan + enhancement) → preparation 상태 inspect → denomination transaction advance → operation end
  • operation은 sync와 advance 전체에서 전용 cancel token과 desired mode를 공유한다.
  • foreground cancelFullSync()는 migration preparation을 취소하지 않는다.
  • foreground와 native 경로는 실제 sync 실행 동안의 SYNC_RUNNING만 공유한다.
  • expiration, foreground handoff, account mutation quiesce는 같은 preparation operation을 취소한다.
  • 일반 status 조회는 task를 제출하지 않는다. startup, unlock, foreground 복귀, mutation 복구 같은 명시적 lifecycle 지점에서만 scheduling을 복구한다.

9.2 Account mutation fence

기존 DB에 계정을 추가하거나 import할 때는 다음 순서를 지킨다.

  1. native migration 작업을 quiesce하고 serial queue가 완전히 비워질 때까지 기다린다.
  2. foreground sync와 mempool observer를 중지한다.
  3. account DB mutation을 실행한다.
  4. foreground sync를 재개한다.
  5. 실제로 resumable manifest가 있을 때만 native preparation task를 다시 제출한다.

마지막 resume는 best-effort recovery다. 이미 commit된 account mutation이나 원래 mutation error를 resume 실패로 덮어쓰면 안 된다.

9.3 Scheduled migration outbox

BackgroundMigrationManager는 com.keplr.vizor.ironwood-migration BGProcessingTask를 관리한다. 준비 task가 proof-ready 상태에 도달한 뒤 생성된 migration outbox만 처리하며 일반 wallet sync를 소유하지 않는다.

9.4 Removed task identifiers

이전 빌드가 제출한 요청이 새 빌드를 깨우지 않도록 앱 시작 시 아래 identifier를 취소한다. handler는 등록하지 않는다.

  • com.keplr.vizor.sync — 제거된 일반 iOS background sync
  • com.keplr.vizor.txtrack — 제거된 legacy transaction tracker
